Spot config drift
before it bites you.

binturong compares your .env files side-by-side in a matrix table. See which keys are missing, empty, or drifted — across every environment at once.

cargo install binturong

Requires Rust · No runtime · Single binary

~/myproject

$ binturong .env .env.staging .env.production

Built for the way you actually work

From local dev to production. From two files to twenty.

🔍

Matrix view

All files side-by-side. All keys in one table. No more juggling multiple diff outputs.

🔎

Auto-discovery

Run binturong with no arguments and it finds all .env* files in the current directory.

🔒

Value masking

Values are hidden as **** by default. Use --values only when you need them.

⚡

CI-friendly

--quiet flag + exit code 1 on drift. Drop it into any pipeline or pre-commit hook.

What each symbol means

Symbol Meaning
✓ **** Key present, value masked
✓ <value> Key present, value visible (with --values)
○ (empty) Key exists but is empty
✗ missing Key not found in this file

Install

$ cargo install binturong

# Then run it anywhere:

$ binturong

$ git clone https://github.com/iamkorun/binturong

$ cd binturong

$ cargo install --path .

exit codes
0 All files in sync
1 Drift detected
2 Error (file not found, unreadable, etc.)
