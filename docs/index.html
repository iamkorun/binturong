made for the messy middle of shipping

from local dev to prod. from two files to twenty. no YAML, no daemon, no SaaS — just one command that refuses to lie to you.

▦

one matrix, every file

every file a column, every key a row. one glance tells you exactly what drifted — no diffing four files by hand.

⌕

auto-discovery, no config

run binturong with no args and it sniffs out every .env* file in the current directory.

✱

masked by default

values hide behind **** so you can share screenshots safely. opt in to the raw view with --values.

↯

CI-ready on day one

--quiet + exit code 1 on drift. drop it into any pre-commit hook or CI pipeline and your build fails fast.

what each symbol means

✓ **** key present, value masked
✓ <value> key present, value visible (with --values)
○ (empty) key exists but is empty
✗ missing key not found in this file

$ cargo install binturong

$ cd binturong

$ cargo install --path .

exit codes
0all files in sync
1drift detected
2error (file not found, unreadable, etc.)
