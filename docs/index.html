Built for every stage, every env.

Two files or twenty. Local, staging, production, preview branches. One command, one table, one answer.

Read drift like a spreadsheet

Every file is a column. Every key is a row. Stop juggling diff outputs between two terminals — scan an entire environment matrix in one glance.

KEY
.env
.staging
.prod

Zero-argument mode

Run binturong with no flags. It finds every .env* file in your working directory and compares them.

$ binturong
→ found 4 files

Masked by default

Values hide behind ****. Screen-share a drift report without leaking secrets. Opt in to values with --values.

Fails loud in CI

Exit code 1 on drift, 0 on match. Drop it into a GitHub Action, a pre-commit hook, or a bash script — your pipeline breaks before prod does.

- name: Check env drift
  run: binturong --quiet

What each symbol means

Symbol Meaning
✓ **** Key present, value masked
✓ <value> Key present, value visible (with --values)
○ (empty) Key exists but value is empty
✗ missing Key not found in this file

exit codes
0 All files in sync
1 Drift detected
2 Error (file not found, unreadable, etc.)
