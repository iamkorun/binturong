sniff out .env drift
before it bites production.

binturong lays every .env file side-by-side in one matrix table. Missing keys, empty values, silent drift — all visible in one glance.

needs Rust · single binary · no runtime · zero config

built for the way you actually ship

from local dev to prod. from two files to twenty. no YAML, no daemon, no SaaS.

▦

matrix view

every file a column. every key a row. one glance tells you what's drifted.

⌕

auto-discovery

run binturong with no args — it finds every .env* in cwd.

✱

value masking

values hidden as **** by default. opt in with --values.

↯

CI-friendly

--quiet + exit 1 on drift. drop it into any pre-commit or pipeline.

what each symbol means

✓ **** key present, value masked
✓ <value> key present, value visible (with --values)
○ (empty) key exists but is empty
✗ missing key not found in this file

install

$ cargo install binturong

# then run it anywhere:

$ binturong

$ git clone https://github.com/iamkorun/binturong

$ cd binturong

$ cargo install --path .

exit codes
0all files in sync
1drift detected
2error (file not found, unreadable, etc.)
